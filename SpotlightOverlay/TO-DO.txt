BUG FIX/TWEAK
Currently, if there's no existing "stuff" on the overlay (no existing arrow, spotlight, etc.) and I'm working on creating my first thing (ex. partway through creating a spotlight, meaning I did the ctrl+click to start the drag, but haven't finalized it to create the final thing) hitting Esc cancels the in-progress shape, but doesn't exit the overlay. If there are no existing, finalized shapes and I hit Esc, I want that to both cancel that in-progress thing AND close out the overall overlay. The goal of this tweak: I shouldn't have to hit Esc (cancel the in-progress thing) and Esc again (close the overlay, which is empty of anything in it) in order to return to normal.

NEW FEATURE: NUMBERS (STEPS) TOOL
Currently in the toolbar, this is called Numbers. This tool is not yet built out. It should instead be called Steps.

This is about placing a map pin a.k.a. teardrop pointer with a number in it. The idea is I click and release and it creates a "step" - a 1 in a teardrop pointer. I click and release again and it creates a 2. I click and release again and it creates a 3, etc.

When using the teardrop style: On click down, I want it to anchor in place, but I want to be able to drag to set the direction of the teardrop's point. On release, that sets the teardrop point at that location. A preference controls exactly how this works.

Settings for the Steps tool
APPEARANCE
Style
-Font (Font and font size)
-Shape - with a teardrop, without a tear drop (under Style)
-Outline - Toggle on/off (under style)
-Size - Slider
-Color #1 (not sure what to call this) - as in the other settings
-Color #2, which is called Outline color - as in the other settings
